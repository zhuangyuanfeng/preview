#21 合并两个有序链表

输入：l1 = 1→2→4，l2 = 1→3→4　|　输出：1→1→2→3→4→4

dummy（哨兵）
l1（待处理）
l2（待处理）
已接到结果
cur 指针

🎯 核心思想：双指针走两条链表，谁小谁先接到 cur 后面，cur 跟着走。

🛡 哨兵 dummy：避开"空头"特判，最后 return dummy.next 即可。

0初始状态
建一个 dummy 节点，cur 指向 dummy。
1比较 l1.val=1 vs l2.val=1，1 ≤ 1 → 接 l1
cur.next = l1；l1 = l1.next；cur = cur.next
2l1.val=2 vs l2.val=1，1 < 2 → 接 l2
cur 接 l2 节点 1，l2 走到 3，cur 走到新节点
3l1.val=2 vs l2.val=3，2 < 3 → 接 l1
4l1.val=4 vs l2.val=3，3 < 4 → 接 l2
5l1.val=4 vs l2.val=4，4 ≤ 4 → 接 l1
6l1 = null，循环结束。剩余 l2 直接接到 cur 后面
cur.next = l1 || l2　不需要 while 逐个接，链表本身是连着的。最后 return dummy.next

🧱 5 步骨架（伪代码）

1. 建 dummy 节点 + cur 指针指向 dummy

2. while (l1 && l2)：比较 val，cur.next = 较小的那个，被接走的指针往后走，cur 也往后走

3. 循环出来后，至少一个空 → cur.next = l1 || l2（剩下整段一起接）

4. return dummy.next

⚠️ 4 个常见坑

坑 1：忘了 cur = cur.next

cur 原地不动，下一轮 cur.next 又被赋值，链表自我覆盖。比较 + 接节点后必须移 cur。

坑 2：返回 dummy 而非 dummy.next

答案多个 0 头节点。永远 return dummy.next。

坑 3：while 写成 ||

当一个为 null 时 .val 报错。while 必须 &&，循环外才接剩余。

坑 4：剩余链表用 while 逐个接

多写一遍循环，浪费。直接 cur.next = l1 || l2，链表本身后面是连着的。
