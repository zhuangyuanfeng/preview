#142 环形链表 II Medium

Hot 100 · 链表 · 第3周

题目

给定一个链表的头节点 head，返回链表开始入环的第一个节点。如果链表无环，则返回 null。

输入：[3,2,0,-4] pos=1
期望：节点 2（index 1）

先想一想

找到环的入口需要几个阶段？

思路（逐步展开）

点击每一步展开详细说明

1 快慢指针找相遇点（同 #141） ▶

slow 每次走 1 步，fast 每次走 2 步。如果有环，它们一定会在环内某个节点相遇。

如果 fast 走到了 null，说明没有环，直接返回 null。

2 相遇后，一个指针回到 head ▶

找到相遇点后，让一个新指针 ptr 从 head 出发，slow 留在相遇点。

这一步是利用数学推导的结论：从 head 到入口的距离 = 从相遇点到入口的距离。

3 两个指针同速走，再次相遇就是入口 ▶

ptr 和 slow 每次都走 1 步，当它们相遇时，相遇点就是环的入口节点。

返回 ptr（或 slow，此时它们指向同一个节点）。

数学推导

点击展开数学证明（面试高频追问） ▶

设 a = head 到入口的距离，b = 入口到相遇点的距离，c = 环长。

  • slow 走了 a + b 步
  • fast 走了 a + b + nc 步（在环里多绕了 n 圈）
  • 因为 fast = 2 * slow，所以 a + b + nc = 2(a + b)
  • 化简得 a + b = nc
  • 即 a = nc - b = (n-1)c + (c-b)

所以从 head 走 a 步 = 从相遇点走 (c-b) 步，都恰好到达入口。

代码

时间 O(n)，空间 O(1) —— 快慢指针 + 数学推导
function detectCycle(head) {
  let slow = head, fast = head
  while (fast && fast.next) {
    slow = slow.next
    fast = fast.next.next
    if (slow === fast) {        // 找到相遇点
      let ptr = head
      while (ptr !== slow) {     // 同速走找入口
        ptr = ptr.next
        slow = slow.next
      }
      return ptr               // 相遇点即入口
    }
  }
  return null                 // 无环
}

关键提醒

这道题的数学推导是面试高频追问，建议背住 a = (n-1)c + (c-b) 这个结论。

记住两个阶段：第一阶段快慢指针找相遇点，第二阶段从 head 和相遇点同速走找入口。推导的核心是 fast 走的路程是 slow 的两倍。
