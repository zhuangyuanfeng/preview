#141 环形链表 Easy

Hot 100 · 链表 · 第3周

题目

给你一个链表的头节点 head，判断链表中是否有环。如果链表中存在环，返回 true；否则返回 false。

输入：head = [3,2,0,-4], pos = 1 // 尾连接到 index 1
输出：true

先想一想

怎么判断链表有环？

思路（逐步展开）

点击每一步展开详细说明

1 快指针走2步，慢指针走1步 ▶

slow 每次走 1 步，fast 每次走 2 步。两个指针都从 head 出发。

2 如果有环，快慢指针一定相遇 ▶

进入环后，快指针每一步都比慢指针多走 1 步，所以它们之间的距离每一步缩小 1，最终一定会相遇。

3 如果 fast 到 null，说明无环 ▶

如果链表没有环，fast 最终会走到 null，循环结束，返回 false。

关键洞察

为什么快慢指针一定相遇？

因为进环后，每一步快指针和慢指针的距离缩小 1。假设进环时两者相距 d 步，那么经过 d 步后距离变为 0，即相遇。

快慢指针代码

时间 O(n)，空间 O(1) —— 面试首选写法
function hasCycle(head) {
  let slow = head
  let fast = head
  while (fast && fast.next) {
    slow = slow.next           // 慢指针走1步
    fast = fast.next.next      // 快指针走2步
    if (slow === fast) return true
  }
  return false
}

Hash Set 版本

点击展开 Hash Set 版（O(n) 空间） ▶
时间 O(n)，空间 O(n) —— 思路更直观但空间开销大
function hasCycle(head) {
  const seen = new Set()
  let curr = head
  while (curr) {
    if (seen.has(curr)) return true
    seen.add(curr)
    curr = curr.next
  }
  return false
}

关键提醒

while 条件是 fast && fast.next，因为 fast 走两步，必须确保 fast 和 fast.next 都不是 null。

如果只检查 fast.next，当 fast 本身是 null 时就会报错。两个条件缺一不可。

今晚练习

  • 快慢指针版：5 分钟盲写，注意 while 条件
  • Hash Set 版：理解思路即可，能口述过程就行
