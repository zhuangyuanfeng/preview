#21 合并两个有序链表 Easy

Hot 100 · 链表 · 第3周

题目

将两个升序链表合并为一个新的升序链表并返回。

输入：l1 = [1,2,4]，l2 = [1,3,4]
输出：[1,1,2,3,4,4]

先想一想

合并两个有序链表，最简单的方式是什么？

思路（逐步展开）

点击每一步展开详细说明

1 创建虚拟头节点 dummy ▶

const dummy = { next: null }

dummy 是一个"假的"头节点，最终返回 dummy.next 就是合并后的链表头。使用 dummy 可以避免处理头节点的特殊情况。

2 比较两个链表当前节点 ▶

用 while (l1 && l2) 循环，每次比较 l1.val 和 l2.val。

两个链表都还有节点时，才需要比较。任何一个走完了，循环就结束。

3 较小的接到结果链表 ▶

如果 l1.val <= l2.val，就把 l1 接到 curr 后面，然后 l1 = l1.next；否则接 l2。

每次接完一个节点后，curr = curr.next，让 curr 指向结果链表的末尾。

4 处理剩余节点 ▶

curr.next = l1 || l2

循环结束后，l1 和 l2 中最多还有一个没走完。因为剩余部分本身就是有序的，直接接到结果链表末尾即可。

模拟过程

以 l1=[1,2,4], l2=[1,3,4] 为例，跟着走一遍

阶段 比较 操作 结果链表 l1 l2
初始 - - dummy [1,2,4] [1,3,4]
Step 1 1 <= 1 接 l1 dummy→1 [2,4] [1,3,4]
Step 2 2 > 1 接 l2 dummy→1→1 [2,4] [3,4]
Step 3 2 <= 3 接 l1 dummy→1→1→2 [4] [3,4]
Step 4 4 > 3 接 l2 dummy→1→1→2→3 [4] [4]
Step 5 4 <= 4 接 l1 dummy→1→1→2→3→4 null [4]
Step 6 - 接剩余 l2 dummy→1→1→2→3→4→4 null null
结束 return dummy.next → 1→1→2→3→4→4

迭代版代码

时间 O(n+m)，空间 O(1) —— 面试首选写法
function mergeTwoLists(l1, l2) {
  const dummy = { next: null }
  let curr = dummy
  while (l1 && l2) {
    if (l1.val <= l2.val) {
      curr.next = l1        // 接较小的
      l1 = l1.next           // l1 前进
    } else {
      curr.next = l2        // 接较小的
      l2 = l2.next           // l2 前进
    }
    curr = curr.next         // 结果链表前进
  }
  curr.next = l1 || l2       // 接上剩余部分
  return dummy.next
}

递归版代码

点击展开递归版（理解即可，不要求盲写） ▶
时间 O(n+m)，空间 O(n+m)（递归栈）—— 理解递归思维
function mergeTwoLists(l1, l2) {
  if (!l1) return l2
  if (!l2) return l1
  if (l1.val <= l2.val) {
    l1.next = mergeTwoLists(l1.next, l2)
    return l1
  } else {
    l2.next = mergeTwoLists(l1, l2.next)
    return l2
  }
}

关键提醒

dummy 节点（虚拟头节点）是链表题的常用技巧，避免处理头节点的特殊情况。

当你不确定最终链表的头节点是谁时，创建一个 dummy 节点，最后返回 dummy.next。这样不需要单独判断"第一个节点怎么接"，代码会简洁很多。
