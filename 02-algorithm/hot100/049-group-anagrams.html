#49 字母异位词分组 Medium

Hot 100 · 哈希表 · 第1周

题目描述

给定一个字符串数组 strs，将所有 字母异位词 组合在一起。可以按任意顺序返回结果。

字母异位词是由重新排列源单词的所有字母得到的新单词。

输入：strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
输出：[["bat"], ["nat", "tan"], ["ate", "eat", "tea"]]

先想一想

如何判断两个单词是异位词？

思路分析（逐步展开）

点击每一步展开详细说明

1 遍历每个单词，排序得到 key ▶

对每个单词的字母进行排序，排序后的结果就是这组异位词的「身份标识」。

例如："eat"、"tea"、"ate" 排序后都变成 "aet"。

"eat".split('').sort().join('')  // → "aet"
"tea".split('').sort().join('')  // → "aet"
"tan".split('').sort().join('')  // → "ant"
2 用 Map 存储：key → [原始单词列表] ▶

使用哈希表 Map，以排序后的字符串为 key，原始单词数组为 value。

处理 "eat"： key="aet" → Map { "aet": ["eat"] }
处理 "tea"： key="aet" → Map { "aet": ["eat","tea"] }
处理 "tan"： key="ant" → Map { "aet": ["eat","tea"], "ant": ["tan"] }
处理 "ate"： key="aet" → Map { "aet": ["eat","tea","ate"], "ant": ["tan"] }
处理 "nat"： key="ant" → Map { "aet": [...], "ant": ["tan","nat"] }
处理 "bat"： key="abt" → Map { "aet": [...], "ant": [...], "abt": ["bat"] }
3 返回 Map 的所有 values ▶

Map 的每个 value 就是一组异位词。用 [...map.values()] 即可收集所有分组。

return [...map.values()]
// → [["eat","tea","ate"], ["tan","nat"], ["bat"]]

完整代码

function groupAnagrams(strs) {
  const map = new Map()
  for (const str of strs) {
    const key = str.split('').sort().join('')
    if (!map.has(key)) map.set(key, [])
    map.get(key).push(str)
  }
  return [...map.values()]
}

复杂度分析

维度 复杂度 说明
时间 O(n * k log k) n 个字符串，每个长度 k，排序 O(k log k)
空间 O(n * k) Map 存储所有字符串

其中 n = 数组长度，k = 最长字符串的长度

进阶思考

如果不用排序，还有什么方法做 key？

用26个字母的计数作为 key！

例如 "eat" 可以编码为 "1#0#0#0#1#...#1#0#..."（a 出现 1 次，e 出现 1 次，t 出现 1 次）。

function groupAnagrams(strs) {
  const map = new Map()
  for (const str of strs) {
    const count = new Array(26).fill(0)
    for (const c of str) {
      count[c.charCodeAt(0) - 97]++
    }
    const key = count.join('#')
    if (!map.has(key)) map.set(key, [])
    map.get(key).push(str)
  }
  return [...map.values()]
}

这样时间复杂度变为 O(n * k)，省去了排序的 O(k log k)。

套路总结

通用模式：哈希表分组

哈希表分组是一个通用模式，key 的设计是关键。

  • 本题的 key：排序后的字符串（或字母计数编码）
  • 核心套路：遍历 → 计算 key → 按 key 归类
  • 同类题目：按某种规则对数据分组时，都可以考虑哈希表
