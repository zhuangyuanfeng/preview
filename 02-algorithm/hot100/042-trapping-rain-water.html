题目

#42 接雨水

多个柱子之间填水，逐位置计算

每个位置的水量受左右最高柱子约束

Hard

先想一想

每个位置能接多少水取决于什么？

取决于该位置左边最高柱子和右边最高柱子中较矮的那个，减去当前高度。

water[i] = min(leftMax, rightMax) - height[i]

这就是"木桶原理"——水位由最短板决定。

方法1：预计算数组（容易理解）

展开查看代码与讲解 ▶

思路：分别预计算每个位置的 leftMax 和 rightMax 数组，然后逐位置求水量。

function trap(height) {
  const n = height.length
  const leftMax = new Array(n)
  const rightMax = new Array(n)

  // 从左往右，记录每个位置左侧的最大值
  leftMax[0] = height[0]
  for (let i = 1; i < n; i++)
    leftMax[i] = Math.max(leftMax[i-1], height[i])

  // 从右往左，记录每个位置右侧的最大值
  rightMax[n-1] = height[n-1]
  for (let i = n-2; i >= 0; i--)
    rightMax[i] = Math.max(rightMax[i+1], height[i])

  // 每个位置的水量 = min(leftMax, rightMax) - height[i]
  let water = 0
  for (let i = 0; i < n; i++)
    water += Math.min(leftMax[i], rightMax[i]) - height[i]

  return water
}
时间 O(n) 空间 O(n)

三次遍历，使用两个额外数组。逻辑清晰，适合理解核心思路。

方法2：双指针（最优）

展开查看代码与讲解 ▶

思路：用左右两个指针从两端向中间逼近，同时维护 leftMax 和 rightMax，不需要额外数组。

function trap(height) {
  let left = 0, right = height.length - 1
  let leftMax = 0, rightMax = 0
  let water = 0

  while (left < right) {
    if (height[left] < height[right]) {
      leftMax = Math.max(leftMax, height[left])
      water += leftMax - height[left]
      left++
    } else {
      rightMax = Math.max(rightMax, height[right])
      water += rightMax - height[right]
      right--
    }
  }

  return water
}
时间 O(n) 空间 O(1)

一次遍历，只用常数空间。面试首选方案。

深入理解

双指针为什么能保证正确？移动较矮一侧的原理是什么？

如果 height[left] < height[right]，那么 left 位置的水量只取决于 leftMax。

为什么？因为右边一定有一个柱子（至少是 height[right]）比 leftMax 高或相等，所以 rightMax >= height[right] >= leftMax。

min(leftMax, rightMax) = leftMax（因为 rightMax 一定更大）
所以可以安全地计算 left 位置的水量：leftMax - height[left]

反之亦然。这就是为什么每次移动较矮一侧不会漏算。
