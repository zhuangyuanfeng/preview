#1 两数之和 Easy

Hot 100 · 哈希表 · 第1周

题目

给定一个整数数组 nums 和一个整数目标值 target，找出数组中和为目标值的两个数的下标。

假设每种输入只会对应一个答案，且同一个元素不能重复使用。

输入：nums = [2, 7, 11, 15], target = 9
输出：[0, 1]
解释：nums[0] + nums[1] = 2 + 7 = 9

先想一想

最容易想到的方法是什么？时间复杂度是多少？

暴力法：两层 for 循环，试遍所有两两组合。

时间复杂度 O(n²)，空间复杂度 O(1)。面试能说但不够。

如何把 O(n²) 优化到 O(n)？用什么数据结构？

哈希表解法（逐步展开）

点击每一步展开详细说明

1 核心思路 ▶

遍历数组，对于每个数 nums[i]，我们要找的是 target - nums[i]。

问题转化为：快速判断一个数之前有没有出现过 → 这正是哈希表擅长的！

2 用 Map 存什么 ▶

Map 存的是：{ 值 → 下标 }

每遍历一个数，先查 Map 里有没有 target - nums[i]：

  • 有 → 找到了！返回两个下标
  • 没有 → 把当前数和下标存进 Map，继续
3 模拟执行过程 ▶

nums = [2, 7, 11, 15], target = 9

i=0 nums[0]=2, 找 9-2=7 Map 没有 7 Map: {2→0}
i=1 nums[1]=7, 找 9-7=2 Map 有 2！下标是 0 返回 [0, 1]
4 完整代码 ▶ 
function twoSum(nums, target) {
  const map = new Map()
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i]
    if (map.has(complement)) {
      return [map.get(complement), i]
    }
    map.set(nums[i], i)
  }
}

时间 O(n)，空间 O(n)。用空间换时间。

为什么用 Map 不用普通对象？

你觉得用 {} 和用 new Map() 有什么区别？

  • 对象的 key 会被转成字符串，数字 key 可能有隐患
  • map.has() 语义更清晰
  • Map 的性能在频繁增删场景下更好
  • 面试时用 Map 更专业

这题的套路价值

通用模式：空间换时间 + 哈希表加速查找

后面这些题都会用到同样的思路：

  • 字母异位词分组 → 哈希表分组
  • 最长连续序列 → Set 判断存在性
  • 无重复字符的最长子串 → Map 记录位置
