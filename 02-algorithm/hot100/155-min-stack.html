#155 最小栈 Medium

Hot 100 · 栈 · 第4周

题目

设计一个支持 push、pop、top 和在常数时间内检索最小元素的栈。

  • MinStack() —— 初始化栈对象
  • push(val) —— 将元素 val 推入栈
  • pop() —— 删除栈顶元素
  • top() —— 获取栈顶元素
  • getMin() —— 获取栈中最小元素
输入：push(-2), push(0), push(-3), getMin()→-3, pop(), top()→0, getMin()→-2

先想一想

怎么在 O(1) 时间获取栈的最小值？

思路（逐步展开）

点击每一步展开详细说明

1 主栈正常 push/pop ▶

主栈 this.stack 负责正常的入栈出栈操作，和普通栈完全一样。

2 辅助栈同步记录当前状态下的最小值 ▶

辅助栈 this.minStack 的栈顶始终保存当前主栈中所有元素的最小值。

两个栈始终保持同步——同时 push，同时 pop，所以长度始终相等。

3 push 时辅助栈 push min(val, 辅助栈顶) ▶

const min = Math.min(val, minStack栈顶)

新元素和当前最小值比较，取较小的那个压入辅助栈。如果辅助栈为空，直接压入 val。

4 pop 时两个栈同时 pop ▶

主栈和辅助栈同时弹出栈顶元素，辅助栈的新栈顶自然就是 pop 之后的最小值。

这就是辅助栈的精髓：每个位置都记录了"到这个位置为止"的最小值，pop 后自动恢复。

模拟过程

以 push(-2), push(0), push(-3), getMin(), pop(), top(), getMin() 为例

操作 主栈 stack 辅助栈 minStack 返回值
push(-2) [-2] [-2] -
push(0) [-2, 0] [-2, -2] -
push(-3) [-2, 0, -3] [-2, -2, -3] -
getMin() [-2, 0, -3] [-2, -2, -3] -3
pop() [-2, 0] [-2, -2] -
top() [-2, 0] [-2, -2] 0
getMin() [-2, 0] [-2, -2] -2

代码

时间 O(1)（每个操作），空间 O(n)（辅助栈）
class MinStack {
  constructor() {
    this.stack = []
    this.minStack = []
  }
  push(val) {
    this.stack.push(val)
    const min = this.minStack.length === 0
      ? val
      : Math.min(val, this.minStack[this.minStack.length - 1])
    this.minStack.push(min)
  }
  pop() {
    this.stack.pop()
    this.minStack.pop()
  }
  top() {
    return this.stack[this.stack.length - 1]
  }
  getMin() {
    return this.minStack[this.minStack.length - 1]
  }
}

关键提醒

辅助栈的精髓是"同步"。

主栈每次操作辅助栈也操作，这样辅助栈顶始终是当前状态的最小值。pop 时不需要重新计算，因为上一个状态的最小值已经记录在辅助栈的下一层了。

这道题考的是数据结构设计思维。

面试中经常作为栈的进阶题出现。核心思路是用空间换时间——额外维护一个辅助栈，把 getMin 从 O(n) 降到 O(1)。
