#206 反转链表 Easy

Hot 100 · 链表 · 第3周

题目

反转一个单链表。

输入：1 → 2 → 3 → 4 → 5
输出：5 → 4 → 3 → 2 → 1

链表基础

链表节点结构：

{ val: 数据, next: 指向下一个节点的指针 }

和数组的区别：

  • 不能随机访问（没有下标），只能从头一个一个遍历
  • 但插入和删除操作是 O(1)（只要知道位置，改指针就行）
  • 数组插入删除是 O(n)（需要移动后面所有元素）

先想一想

反转链表需要几个指针变量？分别有什么作用？

思路（逐步展开）

点击每一步展开详细说明

1 初始化指针 ▶

prev = null，curr = head

prev 是"已反转部分"的头，一开始还没反转任何节点，所以是 null。

2 暂存下一个节点 ▶

next = curr.next

因为下一步要改 curr.next 的指向，所以必须先把原来的 next 存起来，否则链就断了。

3 反转指针 ▶

curr.next = prev

这是核心操作！把当前节点的 next 从"指向后面"改为"指向前面"。

4 移动指针 ▶

prev = curr，curr = next

prev 和 curr 都向前走一步，准备处理下一个节点。

5 循环 & 返回 ▶

重复步骤 2-4，直到 curr === null（所有节点都处理完了）。

最后返回 prev，它就是新链表的头节点。

模拟过程

以链表 1→2→3 为例，跟着走一遍

阶段 next 操作后链表 prev curr
初始 - 1→2→3 null 1
Step 1 2 1→null 1 2
Step 2 3 2→1→null 2 3
Step 3 null 3→2→1→null 3 null
结束 return prev → 3→2→1→null

迭代版代码

时间 O(n)，空间 O(1) —— 面试首选写法
function reverseList(head) {
  let prev = null
  let curr = head
  while (curr) {
    const next = curr.next  // 暂存
    curr.next = prev         // 反转
    prev = curr              // 前进
    curr = next              // 前进
  }
  return prev
}

递归版代码

点击展开递归版（理解即可，不要求盲写） ▶
时间 O(n)，空间 O(n)（递归栈）—— 理解递归思维
function reverseList(head) {
  if (!head || !head.next) return head
  const newHead = reverseList(head.next)
  head.next.next = head  // 让下一个节点指回自己
  head.next = null       // 断开原来的正向链接
  return newHead
}

递归理解检验

递归版中 head.next.next = head 是什么意思？

head.next 是原链表中 head 的下一个节点。

head.next.next = head 就是让这个"下一个节点"的 next 指回 head，实现反转。

然后 head.next = null 断开原来的正向链接，防止出现环。

举例：处理节点 4 时，4.next 是 5，让 5.next = 4（反转），再让 4.next = null（断开）。

关键提醒

链表题的核心就是"画图"。

面试时一定要在纸上画出节点和指针的变化过程，不要只在脑子里想。每一步操作后，箭头指向哪里、哪些链断了，画清楚就不容易出错。
