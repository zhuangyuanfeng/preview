Promise 状态机

三个核心概念

1
Promise 有三种状态：pending → fulfilled 或 pending → rejected
2
状态变化不可逆——一旦从 pending 变成 fulfilled 或 rejected，就不能再变了
3
executor 函数是同步执行的——new Promise(executor) 时，executor 立即被调用

第一步基础骨架

展开查看代码

▶ 
class MyPromise {
  constructor(executor) {
    this.state = 'pending'
    this.value = undefined
    this.reason = undefined

    const resolve = (value) => {
      if (this.state === 'pending') {
        this.state = 'fulfilled'
        this.value = value
      }
    }

    const reject = (reason) => {
      if (this.state === 'pending') {
        this.state = 'rejected'
        this.reason = reason
      }
    }

    try {
      executor(resolve, reject)
    } catch (error) {
      reject(error)
    }
  }
}
关键点：executor 在 constructor 中同步执行，resolve/reject 通过闭包访问实例的 state。try-catch 捕获 executor 执行中的异常。
思考题：为什么 resolve/reject 要检查 state === 'pending'？
答案：
保证状态不可逆，只能从 pending 变化一次。如果不检查，连续调用 resolve 再 reject，状态会被覆盖，违反 Promise 规范。

第二步实现 then 方法

展开查看代码

▶ 
then(onFulfilled, onRejected) {
  if (this.state === 'fulfilled') {
    onFulfilled(this.value)
  }
  if (this.state === 'rejected') {
    onRejected(this.reason)
  }
}
这个版本有问题！
如果 executor 里是异步的（比如 setTimeout 里调用 resolve），then 执行时 state 还是 pending，回调就丢了。 
// 这种情况下 then 的回调不会被执行：
const p = new MyPromise((resolve) => {
  setTimeout(() => {
    resolve('hello')  // 1秒后才resolve
  }, 1000)
})

p.then(val => console.log(val))
// then 执行时 state 还是 pending，两个 if 都不满足

第三步支持异步——回调存储

展开查看代码（完整版）

▶ 
class MyPromise {
  constructor(executor) {
    this.state = 'pending'
    this.value = undefined
    this.reason = undefined
    this.onFulfilledCallbacks = []
    this.onRejectedCallbacks = []

    const resolve = (value) => {
      if (this.state === 'pending') {
        this.state = 'fulfilled'
        this.value = value
        this.onFulfilledCallbacks.forEach(fn => fn())
      }
    }

    const reject = (reason) => {
      if (this.state === 'pending') {
        this.state = 'rejected'
        this.reason = reason
        this.onRejectedCallbacks.forEach(fn => fn())
      }
    }

    try {
      executor(resolve, reject)
    } catch (error) {
      reject(error)
    }
  }

  then(onFulfilled, onRejected) {
    if (this.state === 'fulfilled') {
      onFulfilled(this.value)
    }
    if (this.state === 'rejected') {
      onRejected(this.reason)
    }
    if (this.state === 'pending') {
      this.onFulfilledCallbacks.push(() => onFulfilled(this.value))
      this.onRejectedCallbacks.push(() => onRejected(this.reason))
    }
  }
}
核心改进：
1. 新增两个回调数组 onFulfilledCallbacks 和 onRejectedCallbacks
2. then 中如果 state 是 pending，把回调存到数组里
3. resolve/reject 执行时，遍历数组依次触发回调
思考题：为什么用数组存回调而不是单个变量？
答案：
因为同一个 promise 可以多次调用 then，每次 then 都会注册一个新的回调。如果用单个变量，后注册的回调会覆盖前面的。
const p = new MyPromise(resolve => setTimeout(resolve, 1000))
p.then(() => console.log('回调1'))  // 不能丢
p.then(() => console.log('回调2'))  // 也不能丢

面试回答模板

"Promise 的核心是一个状态机。constructor 接收 executor 并同步执行，内部提供 resolve 和 reject 两个函数来改变状态。then 方法根据当前状态决定是立即执行回调还是存储回调等待状态变化时触发。"
