事件循环 Event Loop

第1周 · JavaScript 深度 · 碎片时间阅读版

全景流程图

一句话概念

JavaScript 是单线程的，事件循环是它处理异步任务的机制——决定了代码的执行顺序。

核心模型

执行规则

  1. 执行同步代码（调用栈）
  2. 调用栈空了 → 清空所有微任务
  3. 微任务清空了 → 取一个宏任务执行
  4. 回到第 2 步，循环往复
关键：微任务比宏任务优先，而且是全部清空，不是只取一个。

微任务 vs 宏任务

微任务 Microtask宏任务 Macrotask
Promise.then / catch / finallysetTimeout / setInterval
await 后面的代码DOM 事件回调（click 等）
MutationObserverrequestAnimationFrame
queueMicrotask()I/O 操作

关键理解

new Promise 的构造函数是同步的

new Promise((resolve) => {
  console.log('我是同步的！')  // 立即执行
  resolve()
}).then(() => {
  console.log('我是微任务')  // 微任务队列
})

await 等价于 .then

// 这两段代码完全等价：
await bar()
console.log('后面的代码')

// 等价于
bar().then(() => {
  console.log('后面的代码')
})
await 后面的代码 = 放进了微任务队列

练习题（输入你的答案）

第 1 题 / 5 入门 
console.log('1')

setTimeout(() => {
  console.log('2')
}, 0)

Promise.resolve().then(() => {
  console.log('3')
}).then(() => {
  console.log('4')
})

console.log('5')
正确答案：1 5 3 4 2
1. console.log('1') → 同步 → 输出 1
2. setTimeout → 回调进宏任务队列
3. Promise.then → 回调进微任务队列
4. console.log('5') → 同步 → 输出 5
5. 清微任务 → 第一个 .then → 输出 3，第二个 .then 入队
6. 继续清微任务 → 第二个 .then → 输出 4
7. 取宏任务 → setTimeout → 输出 2
第 2 题 / 5 入门

⚠️ 注意 new Promise 的构造函数

setTimeout(() => {
  console.log('A')
}, 0)

new Promise((resolve) => {
  console.log('B')
  resolve()
}).then(() => {
  console.log('C')
})

console.log('D')
正确答案：B D C A
1. setTimeout → 回调进宏任务
2. new Promise(executor) → executor 是同步执行的 → 输出 B
3. .then → 回调进微任务
4. console.log('D') → 同步 → 输出 D
5. 清微任务 → 输出 C
6. 取宏任务 → 输出 A
易错点：new Promise 的参数函数是同步执行的！只有 .then 才是微任务。
第 3 题 / 5 进阶

async / await

async function foo() {
  console.log('foo start')
  await bar()
  console.log('foo end')
}

async function bar() {
  console.log('bar')
}

console.log('script start')
foo()
console.log('script end')
正确答案：script start → foo start → bar → script end → foo end
1. console.log('script start') → 同步 → 输出 script start
2. 调用 foo()，进入函数：
• console.log('foo start') → 同步 → 输出 foo start
• await bar() → 先同步执行 bar() → 输出 bar
• await 后面的 console.log('foo end') 变成微任务
3. console.log('script end') → 同步 → 输出 script end
4. 清微任务 → 输出 foo end
await xxx 后面的代码 = 放在 xxx.then() 的回调里 = 微任务
第 4 题 / 5 进阶

嵌套 Promise

Promise.resolve().then(() => {
  console.log('1')
  Promise.resolve().then(() => {
    console.log('2')
  })
}).then(() => {
  console.log('3')
})
正确答案：1 2 3
1. 外层 .then 进微任务队列 → 队列：[外层then]
2. 执行外层 then → 输出 1
• 内层 Promise.then 入队 → 队列：[内层then]
• 外层第二个 .then 入队 → 队列：[内层then, 外层第二个then]
3. 继续清微任务 → 内层 then → 输出 2
4. 继续清微任务 → 外层第二个 then → 输出 3
微任务中产生的新微任务，在当前这轮就会执行，不会等到下一轮。
第 5 题 / 5 面试难度

综合题

console.log('1')

setTimeout(() => {
  console.log('2')
  Promise.resolve().then(() => {
    console.log('3')
  })
}, 0)

Promise.resolve().then(() => {
  console.log('4')
  setTimeout(() => {
    console.log('5')
  }, 0)
})

console.log('6')
正确答案：1 6 4 2 3 5
1. console.log('1') → 同步 → 输出 1
2. 第一个 setTimeout → 回调进宏任务 → 宏：[timer1]
3. Promise.then → 回调进微任务 → 微：[p1]
4. console.log('6') → 同步 → 输出 6
5. 清微任务 → 执行 p1 → 输出 4，setTimeout 进宏 → 宏：[timer1, timer2]
6. 取宏任务 timer1 → 输出 2，Promise.then 进微 → 微：[p2]
7. 清微任务 → 执行 p2 → 输出 3
8. 取宏任务 timer2 → 输出 5
每执行完一个宏任务，都要先清空微任务，再取下一个宏任务。

面试回答模板

"JavaScript 是单线程语言，通过事件循环处理异步。执行顺序是：先执行同步代码，同步代码执行完后清空微任务队列（Promise.then、await 后面的代码），微任务全部清空后再取一个宏任务（setTimeout、I/O）执行。每执行完一个宏任务，都要再检查并清空微任务。这就是为什么 Promise 总是比 setTimeout 先执行。"
追问 async/await 时：
"await 本质上是 Promise 的语法糖。await xxx 后面的代码相当于放在 xxx.then() 的回调里，所以是微任务。"

Node.js 的差异（了解即可）

面试如果追问 Node.js 事件循环和浏览器的区别，只需要说：

  • process.nextTick 优先级高于 Promise.then
  • setImmediate 在 check 阶段执行，和 setTimeout 的先后顺序在不同场景下可能不同
  • 大部分情况下和浏览器表现一致
