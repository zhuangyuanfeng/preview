手写 Promise.all + instanceof

JavaScript 深度 · 手写高频题

Part 1: Promise.all

核心概念：接收一个 Promise 数组，全部 resolve 才 resolve（结果数组顺序和输入一致），任一 reject 就 reject。

查看实现代码 
function promiseAll(promises) {
  return new Promise((resolve, reject) => {
    const results = []
    let count = 0
    promises.forEach((p, i) => {
      Promise.resolve(p).then(val => {
        results[i] = val  // 用 i 保证顺序
        count++
        if (count === promises.length) resolve(results)
      }).catch(reject)
    })
  })
}

为什么用 results[i] = val 而不是 results.push(val)？

push 不保证顺序。因为 Promise 是异步的，完成顺序不确定。比如第 3 个 Promise 可能最先 resolve，用 push 的话它就排到第一个了。用索引 i 才能保证结果顺序和输入一致。

为什么用 count 而不是 results.length 判断完成？

results[i] = val 是稀疏赋值。如果最后一个 Promise 先完成，比如 results[4] = val，此时 results.length 就等于 5，但前面 0~3 还是空位。count 才是真正完成的数量。

为什么要 Promise.resolve(p) 包一层？

输入数组可能包含非 Promise 值。比如 [1, 2, fetch('/api')] 里面有普通数字，Promise.resolve 确保统一处理——非 Promise 值会被自动包装成已 resolve 的 Promise。

Part 2: instanceof

核心概念：判断对象的原型链上是否有构造函数的 prototype。
a instanceof B 就是看 a.__proto__.__proto__... 能不能找到 B.prototype。

查看实现代码 
function myInstanceof(obj, constructor) {
  if (obj === null || typeof obj !== 'object') return false
  let proto = Object.getPrototypeOf(obj)
  while (proto !== null) {
    if (proto === constructor.prototype) return true
    proto = Object.getPrototypeOf(proto)
  }
  return false
}
关键思路：沿着 obj 的原型链一直往上找，每一层和 constructor.prototype 比较。找到就返回 true，到顶（null）还没找到就返回 false。

为什么用 Object.getPrototypeOf 而不是 __proto__？

__proto__ 是非标准属性（虽然浏览器都支持），Object.getPrototypeOf 是 ES5 标准方法。面试中用标准方法更专业，也更符合规范。

面试话术

当面试官问「手写 Promise.all」

"Promise.all 接收一个 Promise 数组，返回一个新 Promise。

核心实现就三点：
1. 用 Promise.resolve(p) 包一层，兼容非 Promise 值；
2. 用索引赋值 results[i] 而不是 push，保证结果顺序和输入一致；
3. 用计数器 count 判断是否全部完成，不能用 results.length，因为稀疏数组的 length 不准。

任一 reject 直接 reject 整个 Promise。"

用 Object.getPrototypeOf 而不是 __proto__，因为前者是标准方法。"
