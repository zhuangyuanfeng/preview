防抖与节流执行时机对比

概念对比

防抖 Debounce

最后一次触发后，等待 delay 再执行。如果在 delay 内再次触发，重新计时。

类比：电梯关门 —— 有人进来就重新等

节流 Throttle

每隔 delay 执行一次，无论触发多少次。固定频率执行。

类比：技能冷却 —— CD 好了才能再放

使用场景

搜索框实时搜索用防抖还是节流？滚动加载用哪个？

搜索框 = 防抖：等用户停止输入后再发请求，避免每按一个键都请求一次。
滚动加载 = 节流：固定频率检查滚动位置，不需要每一像素都计算。

更多场景：
防抖：窗口 resize、表单验证、按钮防重复点击
节流：mousemove 画图、拖拽、射击游戏开枪

防抖实现

Step 1：基础版防抖

展开代码 ▼
核心思路：每次触发都清除上一个定时器，重新设定。只有最后一次触发后等够 delay 才执行。
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}
要点：
1. 闭包保存 timer 变量
2. clearTimeout + 重新 setTimeout 实现"重新计时"
3. fn.apply(this, args) 保证 this 指向和参数传递正确

Step 2：带 immediate（leading）参数

展开代码 ▼ 
function debounce(fn, delay, immediate = false) {
  let timer = null
  return function(...args) {
    if (immediate && !timer) {
      fn.apply(this, args)
    }
    clearTimeout(timer)
    timer = setTimeout(() => {
      timer = null
      if (!immediate) fn.apply(this, args)
    }, delay)
  }
}

immediate 参数有什么用？

第一次触发立即执行，后续在 delay 内的触发被忽略。

典型场景：防止按钮重复点击 —— 第一次点击立即提交，之后连续点击无效，直到冷却结束。

实现关键：用 !timer 判断是否是"第一次"（冷却结束后 timer 被设为 null，又可以立即执行了）。

节流实现

Step 1：时间戳版（leading）

展开代码 ▼
核心思路：记录上次执行时间，当前时间减去上次时间 >= delay 就执行。
function throttle(fn, delay) {
  let lastTime = 0
  return function(...args) {
    const now = Date.now()
    if (now - lastTime >= delay) {
      fn.apply(this, args)
      lastTime = now
    }
  }
}
特点：第一次触发立即执行（因为 lastTime 初始为 0），最后一次触发不保证执行。

Step 2：定时器版（trailing）

展开代码 ▼ 
function throttle(fn, delay) {
  let timer = null
  return function(...args) {
    if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args)
        timer = null
      }, delay)
    }
  }
}

时间戳版和定时器版有什么区别？

时间戳版（leading）：
· 第一次触发立即执行
· 最后一次触发不执行（如果距离上次执行不够 delay）

定时器版（trailing）：
· 第一次触发delay 后才执行
· 最后一次触发也会执行（timer 到期后执行）

面试加分项：可以结合两种方式，实现 leading + trailing 的完整版节流。

面试话术

参考回答
防抖和节流都是优化高频事件的手段。

防抖的核心是"延迟执行"——每次触发都重置计时器，只有停止触发后等够延迟才真正执行。适用于搜索输入、窗口 resize 等场景。实现上用闭包保存 timer，每次 clearTimeout 再重新 setTimeout。

节流的核心是"固定频率"——不管触发多频繁，都按固定间隔执行一次。适用于滚动监听、mousemove 等场景。有时间戳和定时器两种实现，时间戳版首次立即触发（leading），定时器版最后一次也能触发（trailing）。

在实际项目中，我会根据业务需要选择：需要即时反馈用节流，需要最终态用防抖。
