Part 1 · 柯里化 (Currying)

柯里化：把接受多个参数的函数转换为一系列接受单个参数的函数。
add(1, 2, 3) → curry(add)(1)(2)(3)
实现 · curry 函数 
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args)
    }
    return function(...newArgs) {
      return curried.apply(this, [...args, ...newArgs])
    }
  }
}
核心思路：递归收集参数，当已收集的参数数量 >= fn.length（形参个数）时执行原函数，否则返回新函数继续收集。
fn.length 是什么？
函数的形参个数。curry 通过比较已收集参数数量和形参数量来判断是否该执行。注意：rest 参数、默认值参数不计入 fn.length。
使用示例 
const add = (a, b, c) => a + b + c
const curriedAdd = curry(add)

curriedAdd(1)(2)(3)    // 6
curriedAdd(1, 2)(3)    // 6
curriedAdd(1)(2, 3)    // 6
柯里化的妙处在于灵活组合——可以一次传一个参数，也可以一次传多个，只要总数够了就执行。

Part 2 · 发布订阅 (EventEmitter)

发布订阅模式：一个事件中心，允许订阅(on)、发布(emit)、取消订阅(off)。
Vue 的 $emit/$on、Node 的 EventEmitter 都是这个模式。
重点 实现 · EventEmitter 
class EventEmitter {
  constructor() {
    this.events = {}
  }

  on(event, fn) {
    if (!this.events[event]) this.events[event] = []
    this.events[event].push(fn)
  }

  off(event, fn) {
    if (!this.events[event]) return
    this.events[event] = this.events[event].filter(f => f !== fn)
  }

  emit(event, ...args) {
    if (!this.events[event]) return
    this.events[event].forEach(fn => fn(...args))
  }

  once(event, fn) {
    const wrapper = (...args) => {
      fn(...args)
      this.off(event, wrapper)
    }
    this.on(event, wrapper)
  }
}
once 为什么要包一层 wrapper？
需要在执行后 off 掉自己，但 off 比较的是函数引用。如果直接 off(fn) 会找不到——因为 on 注册的是 wrapper 而不是 fn。所以必须 off(event, wrapper)，而这就要求我们用一个 wrapper 把 fn 包起来。

面试话术

当面试官问「手写柯里化」

「柯里化的核心是递归收集参数。返回的 curried 函数每次被调用时，把新参数追加到已有参数上。当参数总数达到原函数的形参数量（fn.length），就执行原函数；否则返回新函数继续等待参数。」

当面试官问「手写发布订阅」

「发布订阅本质是一个事件中心，内部用一个对象维护事件名到回调数组的映射。on 就是往数组 push，emit 就是遍历数组执行，off 就是 filter 掉对应引用。once 的关键是用 wrapper 包装回调，执行后立即 off(wrapper)。」

加分点

  • 柯里化：提到它和偏函数（partial application）的区别
  • 柯里化：提到 fn.length 对 rest 参数和默认值的限制
  • 发布订阅：提到与观察者模式的区别（有无事件中心 / 中间层）
  • 发布订阅：提到内存泄漏问题——忘记 off 会导致回调一直留在内存中
