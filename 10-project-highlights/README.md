# 项目亮点包装

> 优先级：⭐⭐⭐⭐⭐ | 建议时间：第10-11周

## 包装原则
- 用 STAR 法则：Situation → Task → Action → Result
- 量化结果：性能提升 X%、效率提升 X%、覆盖 X 个业务
- 突出技术深度，不只是"做了什么"，而是"为什么这么做"

## 需要准备的项目（3-4个）

### 项目1：电商核心业务（已有经验）
- [ ] 整理最有技术深度的业务场景
- [ ] 性能优化的具体数据
- [ ] 遇到的技术难点与解决方案

### 项目2：工程化/基建（如果有）
- [ ] 构建优化、CI/CD、组件库等
- [ ] 提效数据

### 项目3：AI 相关项目（新做）
- [ ] AI Chatbot 或 RAG 应用
- [ ] 技术选型理由
- [ ] 遇到的挑战（流式渲染、Token 优化等）

### 项目4：开源/Side Project（加分）
- [ ] GitHub 上可展示的项目
- [ ] npm 包或工具

## 简历优化
- [ ] 用技术语言重写项目描述
- [ ] 突出 AI 前端经验
