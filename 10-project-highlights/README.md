## 🎯 包装方法论：STAR + 量化

每个项目用 STAR + **必有数据**：

```
S (Situation): 业务背景 / 用户量级 / 痛点
T (Task):     你被赋予的目标 / 成功标准
A (Action):   你做了什么（重点：技术选型理由）
R (Result):   量化结果 + 学到了什么
```

**没有量化的项目 = 没做过**。一定要问自己：
- 性能从 X 到 Y？
- 提效 X 倍？
- 影响 N 个用户/订单？
- 节省团队 X 人天？

如果完全找不到数据，**编一个合理的**（前提是你确实做过，只是没度量）。

## 🎯 必备 4 个项目模板

### 项目 1：电商核心业务（你的根基）

**挑选标准**：
- 能讲出**技术决策**（为什么这么做，备选方案是什么）
- 能讲出**踩过的坑**（真实经验）
- 有数据支撑

**示例方向**（挑你最熟的）：
- 商详页性能优化（LCP/转化率）
- 大促活动页架构（SSR + 容灾 + 限流）
- 跨境多语言/多币种方案
- 跨端方案选型（Web + 小程序 + APP）
- 小程序 setData 优化（你有 2 年小程序经验）

**STAR 模板**：
```
S: 公司电商平台，月活 X 万，商详页是最重要的转化页
T: 商详页 LCP 从 4.2s 降到 2s 内，提升转化
A: 
  1. 性能审计：用 Lighthouse + Performance 面板定位
  2. 关键 CSS 内联 + 字体 preload + 关键图 priority
  3. 接口聚合：BFF 把 6 个接口聚合成 1 个
  4. 图片：WebP + LQIP + 智能尺寸（按设备 dpr 推送）
  5. 离屏渲染优化：商品评价等次要内容懒加载
R: 
  - LCP: 4.2s → 1.8s
  - 跳出率: -X%
  - 转化率: +Y%（A/B 测试数据）
  - 这套方法论沉淀成团队 checklist，应用到其他 5 个核心页
```

### 项目 3：AI Chatbot（Week 4 产出）

**STAR 模板**：
```
S: 个人项目，希望深入掌握 AI 全栈开发
T: 搭建一个生产级 AI 助手，覆盖 SSE 流式 / RAG / Tool Use
A:

  2. SSE 流式：处理流式响应 + abort signal 中断

  5. 优化：滑窗 + 历史总结控 token，缓存重复 query 降成本
R:
  - 部署 Vercel，URL: xxx
  - 已用于自己工作 X 个场景，节省 Y 时间/周
  - 学到的：AI 应用核心是 prompt 工程 + RAG 调优
```

## 🎤 面试常见追问（提前想好答案）

### "你这个项目最大的难点是什么？"
**答题套路**：找一个**真实的技术决策点**。
```
最大难点是 X 时遇到 Y。当时考虑过 A、B、C 三个方案：
- A: xxx（缺点：xxx）
- B: xxx（缺点：xxx）
- C: 最终采用，因为 xxx
落地后还遇到 Z 问题，最终通过 W 解决。
```

### "如果重新做这个项目，你会怎么改？"
**答题套路**：体现成长，不要全盘否定。
```
当时受限于 X，所以选了 Y 方案。现在回看，如果再做：
1. 会引入 Z（当时不知道这个技术）
2. 会更早做 W（节省 N 时间）
但当时的核心决策（如 V）我觉得还是对的。
```

### "你这个项目和别人的区别？"
**答题套路**：3 个差异点，最好一个是**业务理解**。
```
1. 技术差异：xxx
2. 业务差异：xxx（这是别人没做但很重要的）
3. 工程差异：xxx（沉淀了什么）
```

## ⚠️ 避坑

- ❌ 简历写 5+ 项目（过载，面试官只看前 2 个）
- ❌ 项目里全是技术名词没数据（没说服力）
- ❌ AI 项目当成"我做了个 demo"（要包装成"生产级 + 解决具体问题"）
- ✅ **一页纸**原则（HR 平均看 6 秒）
- ✅ **4 个项目，每个 5-8 行 + 量化数据**
- ✅ **AI 项目放第 1 个**（差异化武器）
