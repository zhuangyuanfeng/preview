# 开始新一天的学习（study-start）

**核心语义**：今天**第一次**开学习。引导启动仪式（对抗心流启动困难），明确今日目标。
用户调用此 skill 后**什么都不用做**——Claude 自动拉最新、读上下文、给今日地图。

**与其他 skill 的边界**：

| Skill | 用法 |
|---|---|
| `/study-start`（本 skill）| 新的一天，第一次开学，需要启动仪式 |
| `/study-context` | 中途从另一台电脑/几小时前继承上下文 |
| `/study-sync` | 切机前同步（不结束当天） |
| `/study-wrap` | 当天真正结束 |

如果用户语境明显是"中途回来"，建议改用 `/study-context`。

---

## 步骤 1：自动 git pull（确保拉到最新）

用 Bash 执行：

```bash
cd ~/preview && git pull --rebase
```

如果失败（冲突/网络/凭据）：
- 报告错误原因
- **不要继续后续步骤**，提示用户手动处理
- 不要自动重试

成功（包括"Already up to date"）就继续步骤 2。

## 步骤 2：读核心文件

1. `~/preview/handoff.md` — 看今日 Next Steps（昨天 wrap 时写的）
2. `~/preview/WEEKLY-PLAN.md` — 找当前 Week 卡片
3. `~/preview/PAIN-POINTS.md` — 看本周计划解的痛点

## 步骤 3：问用户两件事（一题一问）

**Q1：你今天有多少时间能学？**
- < 1h（少量）：只做 1 项核心任务
- 1-2h（标准）：handoff Next Steps 的 1-2 项
- 2-3h（充裕）：完整 Day 量 + 痛点项目
- > 3h（高强度，不推荐天天这样）：全量 + 1 个 demo

**Q2：你的精力状态怎么样？**
- 满血：直接进硬任务（算法/手写题）
- 一般：先轻输入（视频/文章）再硬任务
- 疲惫：调整今日目标，不强迫

**关键**：根据 Q1+Q2 给量身建议，不要让用户硬扛。

## 步骤 4：呈现"今日地图"

格式：

```
🌅 今天开学

📍 当前位置
- 阶段：v2 / Week N (MM.DD-MM.DD) — 当周主题
- 累计：算法 X/100 · JS 手写 X/9 · 痛点 X/10
- 上次断点：[handoff 上次的 What I Was Doing]

🎯 今日目标（基于你 X 小时 + Y 状态）
1. [具体任务，含预估用时]
2. [具体任务]
3. [可选/拓展]

⏰ 心流时段建议
- 启动仪式 5 步（见你贴桌面的版本）
- 心流：[建议时段，连续 X-Y 分钟]
- 中途休息：1-2 次，每次 ≤ 10 分钟
- 收工时记得调用 /study-wrap

💡 准备好后回复：
   A. 「开始第 1 步」→ 进入心流
   B. 「先看一下第 1 步的资料」→ 我先讲解
   C. 「调整目标」→ 重新规划
```

## 步骤 5：启动仪式提醒（重要）

如果用户没有桌面贴启动仪式，主动提醒：

> 你之前说要把启动仪式 5 步贴桌面。现在贴了吗？
> 没有的话，先花 2 分钟写完贴上——心流型 + 启动困难的人，仪式比意志力靠谱。
>
> 参考 5 步：
> 1. 倒一杯温水（物理信号）
> 2. 关 IM 通知 30 分钟（环境信号）
> 3. 打开 Week 卡片看 1 分钟（认知信号）
> 4. 听一首固定的歌（条件反射）
> 5. 打开编辑器 → 进入

## 步骤 6：等用户回复后，按选择执行

不要主动开始，让用户确认。

## ❌ 不要做的事

- ❌ 不要假设用户精力满满（先问 Q2）
- ❌ 不要硬塞完整 Day 量给疲惫的用户
- ❌ 不要省略启动仪式提醒（这是 v2 核心）
- ❌ 不要更新 handoff.md（开学不写文件）
- ❌ 不要触发倦怠预警（除非用户主动说累）

## ⚡ 特殊场景

### 周一开学
额外展示本周 Week 卡片（不只是今天）：

```
📅 本周（Week N）总目标
- 算法：+X 题（链表/栈/...）
- 手写题：[本周补的]
- 视觉产出：[本周必出的图]
- 痛点项目：[本周计划解的]
- 周日复盘：5 分钟讲一遍 + 写复盘
```

### Week 5-7（5.26-6.15）开学
额外提醒倦怠风险：

```
⚠️ 今天进入倦怠高发周。
建议把痛点项目排在今天，让 AI 帮你解决一个真实工作痛点——
"立即有用"是抗倦怠最强解药。
```

### 用户连续 ≥ 2 天没学
温和提醒（不指责）：

```
注意到你已经 X 天没学了。先别担心——v2 设计是周里程碑，不是日打卡。
今天能挤多少时间？我帮你定一个**最小可行**目标，今天先把节奏捡回来。
```
