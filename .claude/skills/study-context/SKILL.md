# 恢复学习上下文（study-context）

新 session 启动时调用此 skill，3 步完成上下文恢复。

## 步骤 1：读 4 个核心文件

按顺序 Read：

1. `~/preview/LEARNING-CONTEXT.md` — **静态**：用户画像、学习风格、协作约定、v2 决策、文件导航
2. `~/preview/handoff.md` — **动态**：当前阶段、当前进度、下一步行动（每次同步更新）
3. `~/preview/WEEKLY-PLAN.md` — 找到当前周里程碑卡片（按 handoff.md 里的 Current Phase 定位）
4. `~/preview/PAIN-POINTS.md` — 工作痛点池（用户兴趣型驱动来源，每周必解 1 个）

## 步骤 2：输出"状态恢复 + 下一步引导"（必须两段都有）

⚠️ **关键**：不能只输出状态，否则用户会"不知道下一句话怎么说，感到莫名其妙"。
必须包含 **状态** + **接下来要做的** + **用户的下一步选择** 三段。

完整格式（10-15 行可接受）：

```
✅ 上下文已恢复

📊 状态
- 你：视觉先导 / 全局反思 / 心流型 / 成就驱动（缺兴趣型驱动）
- 阶段：v2 / Week N (MM.DD-MM.DD) — 当周主题
- 进度：算法 X/100 · JS 手写 X/9 · 痛点池 X/10
- 累计学习日：X 天 / 当前连续 X 天

🎯 接下来你要做的（来自 handoff.md → Next Steps）
1. [最近时间线第 1 项，写具体]
2. [第 2 项，写具体]
3. [第 3 项，如有]

💡 你的下一步可以选：
   A. 「开始第 1 步」→ 我陪你跑
   B. 「今天我有 X 小时，最该做啥」→ 我帮你挑
   C. 「我要复盘」/「我要收工」→ 调用 /study-wrap
   D. 直接说你想做的事

约定：一题一问 · 视觉化 · 直白反馈 · 配 Demo · 绝不延期
```

输出后**等待用户回复**，不要主动开始任何任务。

## ❌ 不要做的事

- 不要重新讨论 v1 vs v2 的决策（已经定了，commit `3b35ee1`）
- 不要重新提议做学习风格测评（已经做了，2026-04-27）
- 不要把整个文件复制粘贴给用户（用户已经知道自己写了啥）
- 不要在确认输出里**省掉「下一步选择」段**——用户会因此卡壳
- 不要主动开始第 1 步任务——必须等用户确认

## ✅ 确认后等待用户指令，按以下约定执行

来自 LEARNING-CONTEXT.md 协作约定，必须遵守：

- **视觉先导**：图 / 类比 / 动画优先于纯文字
- **直白反馈**：不用苏格拉底式反问，不让用户猜
- **给独处时间**：讲完留时间消化，不连续追问
- **配 Demo**：每个知识点必须 1 张图 + 1 个 50 行内 demo（防听懂幻觉）
- **一题一问**：测评/问卷类一题一问（用户记忆力受限）
- **熟悉领域允许直接动手**：前端可上手，AI 等陌生领域要原理先行

## 🚨 主动预警时机

发现以下情况主动提醒用户：

- 当前是 **Week 5-7（5.26-6.15）**：倦怠高发期，强调痛点项目刺激度
- 用户连续 ≥ 2 天没碰算法/JS：可能进入"听懂幻觉"陷阱，主动建议出 demo
- 用户提议"延期 / 砍目标 / 改回日打卡"：违反核心约定，温和但坚定指出 v2 的设计原因

## 何时主动调用此 skill

如果新 session 第一句话符合下列模式，且 Claude 还没读过 LEARNING-CONTEXT.md，**自动调用此 skill 而不要等 user 显式说"恢复上下文"**：

- "开始" / "继续" / "接着学" / "我回来了"
- "今天该做什么" / "今晚做什么"
- "Day N" / "Week N"
- "/study-context"（显式调用）
- 任何在 cwd=`~/preview` 下的开场白
