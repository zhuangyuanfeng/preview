## 🎯 自我介绍（必背）

### 1 分钟版（开场用）

```
您好，我是 X，8 年前端开发经验，目前在 [公司] 负责 [核心业务]。

技术上聚焦三块：
1. 电商场景的复杂前端架构（最近做的是 [项目]，[一句话亮点]）
2. 跨端开发，深入做过小程序 + Web，对双线程架构有深度理解
3. 最近 6 个月在做 AI 全栈方向，搭建了 [Chatbot 项目]，部署在 Vercel 上，URL 在简历里

希望加入贵司的原因是 [具体理由，不要套话]。
```

**60 秒能说完，包含：经验、技术栈、AI 差异化、加入理由**。

### 3 分钟版（深度版）

在 1 分钟版基础上加：
- 一个最有成就感的项目（30 秒）
- 一个体现成长的故事（30 秒）
- 对未来 2-3 年的规划（30 秒）

## 🎯 技术高频题（每题准备 1 段标准回答）

### JS 核心 4 题

| 题目 | 关键词 |
|---|---|
| 事件循环完整流程 | 宏任务/微任务/MutationObserver/RAF |
| 闭包 + 内存泄漏 | 词法作用域 / setTimeout 内回调 |
| 原型链 + new 操作符 | Object.create / 4 步 |
| Promise 完整实现 | 状态机 + then 链 + 异步 |

### 框架 3 题

| 题目 | 关键词 |
|---|---|
| Vue3 响应式原理 | Proxy/effect/track/trigger |
| React Hooks 链表设计 | fiber.memoizedState |
| Vue vs React 核心差异 | 响应式 vs 不可变 |

### 网络 3 题

| 题目 | 关键词 |
|---|---|
| HTTP 缓存完整流程 | 强缓存 → 协商缓存 |
| HTTPS 握手 | 非对称交换密钥，对称传数据 |
| CORS 完整机制 | 预检 OPTIONS + 简单请求 |

### AI 3 题（差异化）

| 题目 | 关键词 |
|---|---|
| SSE 流式渲染原理 | text/event-stream + ReadableStream |
| RAG 完整链路 | chunking → embedding → 检索 → 拼 prompt |
| Function Calling | tool_use → 执行 → tool_result 回写 |

每题去 03-08 模块 README 找对应「面试话术」段。

## 🎯 行为面 6 大经典问题（必备）

### Q1: 为什么离职？
**模板**：
```
现在公司业务/技术成长空间和我的目标有差异：
- 业务：[具体差异，不要抱怨]
- 技术：[想往 X 方向深入，但当前业务用不到]
所以希望找一个 [AI 方向 / 全栈方向 / 大厂深度] 的机会。
```
**禁忌**：抱怨老板 / 同事 / 加班 / 薪资。

### Q2: 你最有成就感的项目？
直接对应 `10-project-highlights/` 中的项目 1。

### Q3: 遇到的最大技术挑战？
**模板**：
```
做 [项目] 时遇到 [具体技术问题]，影响 [业务后果]。
排查过程：[假设1 → 验证 → 排除] [假设2 → 定位]
解决方案：[最终方案]，并 [推广到其他场景]
```
**关键**：体现**调试方法论**，不是单纯运气好。

### Q4: 如何看待技术选型？
**模板**：
```
我的选型框架：
1. 业务需求第一（不为新技术而新）
2. 团队能力匹配（学习成本 vs 收益）
3. 长期维护性（社区活跃度 + 文档完善度）
4. 容错（出问题怎么 fallback）

举个例子：[一次真实选型，比如选 Vue 还是 React]
```

### Q5: 团队冲突怎么处理？
**模板**：用具体例子。
```
和 [角色] 在 [问题] 上有分歧。
我的处理：
1. 先理解对方立场（约个会面，听他完整观点）
2. 找共同目标（都希望项目成功）
3. 用数据/事实推进决策（而不是争输赢）
4. 留 paper trail（决策记录文档）
最终结果：[双赢 / 妥协 / 让步但数据证明对方对]
```
**关键**：不要装"我从来不和人冲突"，**真实** + **方法论**。

### Q6: 未来规划？
**模板**：
```
2-3 年：在 AI 全栈方向做 X 个深度项目
3-5 年：成为 [架构师 / 技术 leader / 独立 contributor]
长期：[更长期的方向，体现持续学习]
```
**关键**：和你应聘的岗位一致。

## 🔧 用 AI 辅助模拟面试

**Prompt 模板**（直接拿去用）：

```
你是 [公司] 的资深前端面试官，正在面试一个 8 年前端经验、最近转 AI 全栈方向的候选人。
我的简历亮点是：
- 8 年电商前端
- 跨端深度（小程序 + Web）
- 最近做了 [AI 项目，简介]

请你：
1. 模拟一场 45 分钟技术面
2. 包含：自我介绍 → 技术问题 4-5 题（覆盖 JS/框架/网络/AI）→ 项目深挖 → 反向提问
3. 一题一题问，等我回答后再追问或下一题
4. 面试结束后给我反馈（哪些回答好/差）
```

## ⚠️ 避坑

- ❌ 不要在第一场就投梦想公司（练手提升后再投）
- ❌ 不要每场都"完美"才进下一场（迭代比完美重要）
- ❌ 不要因为一两场拒掉就放弃（前 3-5 场拒是正常的）
- ✅ **每场必复盘**，被问的题归档（可能下家也会问）
- ✅ 反向提问准备 3 个（体现你是有思考的人）
