# 学习能力评估与计划调整

> 每周日裸写检测后，同步更新此文件
> 根据数据动态调整后续计划，不死守原计划

---

## 评估维度

| 维度 | 观察方式 | 预警信号 |
|------|---------|---------|
| 算法理解速度 | 一道新 medium 题从读题到有思路要多久 | > 30min 还没思路 |
| 代码转化能力 | 有思路后写成能跑的代码要多久 | 思路对但写不出来 |
| 记忆保持度 | 裸写检测中，上周做过的题还能写出来吗 | 连续 2 周同题 < 3⭐ |
| 概念吸收速度 | 看完一篇源码文章，能用自己的话讲清吗 | 看了但说不出来 |
| 手写题熟练度 | 高频手写题能否 10min 内盲写 | 超过 15min 或写不对 |
| 疲劳度 | 晚上学习时注意力能集中多久 | 经常不到 1h 就走神 |

## 可能的调整方案

### 如果算法进度慢
- 减少每周新题量（5题 → 3题）
- 增加重复练习（同一题隔天再写一遍）
- 降低难度目标（放弃 hard，专攻 easy + medium）

### 如果记忆衰退快
- 加入间隔重复：当天 → 第3天 → 第7天 → 第14天
- 每题写一句话总结思路（建自己的题解卡片）
- 周中加一次 10min 快速回顾

### 如果源码看不进去
- 换方式：文章 → 视频 → 手写简化版
- 缩小范围：只吃透响应式，编译器和 diff 降为"能说概念"
- 不追求看懂每一行，能画出流程图就够

### 如果晚上精力不够
- 算法挪到早上（早起 30min）
- 晚上只做轻度输入（看文章）
- 周末补完整时段的练习

### 如果整体进度落后
- 优先保算法 + 项目包装（这两个直接决定能不能过面试）
- Vue3 源码缩到只讲响应式
- React 降到"能说差异"就行，不写项目
- AI 项目用更简单的方案（纯 Chat，不做 RAG）
